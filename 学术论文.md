# 基于 React + LangGraph 的可视化 AI 工作流编排系统

---

## 摘要

随着大语言模型技术的快速发展，AI 应用开发的需求日益增长，但复杂工作流的构建仍面临技术门槛高、灵活性不足等挑战。本文设计并实现了一套基于 React + LangGraph 的可视化 AI 工作流编排系统，采用前后端分离架构，后端使用 FastAPI 框架提供 RESTful API，前端通过 React Flow 实现拖拽式节点编辑。系统集成了多种功能节点，包括用户输入、大模型调用、代码执行、知识库检索、条件判断和输出节点，支持动态变量传递和条件分支跳转。知识库模块采用 Milvus 向量数据库，实现了密集向量与稀疏向量的混合检索，支持分数归一化和加权融合。实践表明，该系统能够显著降低 AI 工作流的构建难度，提高开发效率，为各类 AI 应用场景提供灵活、可扩展的解决方案。

**关键词**：人工智能；工作流编排；向量数据库；大语言模型；检索增强生成

---

## Abstract

With the rapid development of large language model technology, the demand for AI application development is growing, but the construction of complex workflows still faces challenges such as high technical thresholds and insufficient flexibility. This paper designs and implements a visual AI workflow orchestration system based on React + LangGraph, adopting a front-end and back-end separation architecture. The back-end uses the FastAPI framework to provide RESTful APIs, while the front-end implements drag-and-drop node editing through React Flow. The system integrates various functional nodes, including user input, large model invocation, code execution, knowledge base retrieval, conditional judgment, and output nodes, supporting dynamic variable passing and conditional branch jumping. The knowledge base module adopts the Milvus vector database, implementing hybrid retrieval of dense and sparse vectors with score normalization and weighted fusion. Practice shows that the system can significantly reduce the difficulty of building AI workflows, improve development efficiency, and provide flexible and scalable solutions for various AI application scenarios.

**Keywords**：Artificial Intelligence; Workflow Orchestration; Vector Database; Large Language Model; Retrieval-Augmented Generation

## 绪论

### 1.1 研究背景与意义

近年来，大语言模型（Large Language Model, LLM）技术取得突破性进展，在自然语言处理、代码生成、知识问答等领域展现出强大能力。然而，将大模型应用于实际业务场景时，往往需要构建复杂的工作流，涉及多步骤处理、条件判断、数据流转等环节。传统的开发方式需要编写大量代码，技术门槛较高，难以满足快速迭代和灵活调整的需求。

可视化工作流编排技术通过图形化界面降低了复杂系统的构建难度，已在数据处理、机器学习等领域得到广泛应用。将可视化编排与大模型技术结合，能够显著提高 AI 应用的开发效率，使非技术人员也能参与到 AI 系统的设计与构建中。

### 1.2 国内外研究现状

在工作流编排领域，Apache Airflow、Prefect 等工具提供了成熟的批处理工作流解决方案，但主要面向数据工程场景，对大模型集成支持不足。LangChain 框架通过链式调用实现了 LLM 应用的快速构建，但缺乏直观的可视化编辑界面。LangGraph 作为 LangChain 的扩展，引入了状态图概念，支持更复杂的工作流逻辑，但其编程式构建方式仍对开发者有一定要求。

在知识库检索方面，基于向量数据库的检索增强生成（Retrieval-Augmented Generation, RAG）技术已成为大模型应用的重要范式。Milvus、Pinecone 等向量数据库提供了高效的向量检索能力，混合检索、重排序等技术进一步提升了检索精度。

### 1.3 本文主要工作

本文设计并实现了一套完整的可视化 AI 工作流编排系统，主要工作包括：

1. 设计前后端分离的系统架构，后端基于 FastAPI 提供 RESTful API，前端基于 React + TypeScript 实现可视化界面。
2. 基于 LangGraph 实现工作流引擎，支持多种节点类型和条件分支逻辑。
3. 集成 Milvus 向量数据库，实现密集向量与稀疏向量的混合检索，支持分数归一化和加权融合。
4. 实现沙箱化代码执行环境，支持 Python 代码的安全执行。
5. 通过 React Flow 实现拖拽式节点编辑，提供友好的用户交互体验。

## 论文主体

### 2.1 系统架构设计

系统采用前后端分离的分层架构，主要包括以下层次：

**前端层**：基于 React 18 + TypeScript 构建，使用 Vite 作为构建工具，Ant Design 提供 UI 组件，React Flow 实现可视化编排，Zustand 管理应用状态。前端负责工作流的可视化编辑、参数配置和执行监控。

**后端层**：基于 FastAPI 框架构建，使用 SQLAlchemy 进行数据库操作，LangGraph 实现工作流引擎。后端提供工作流管理、知识库管理、工作流执行等核心 API。

**数据层**：包括 SQLite/PostgreSQL 关系型数据库和 Milvus 向量数据库。关系型数据库存储工作流定义、知识库元数据等结构化信息；向量数据库存储文档向量，支持高效的语义检索。

### 2.2 工作流引擎设计

工作流引擎基于 LangGraph 的 StateGraph 实现，核心设计包括：

**状态管理**：定义 WorkflowState 数据结构，包含固定字段和动态变量存储区 custom_vars，支持各节点间的数据传递。

**节点类型**：系统支持七种节点类型：
- 输入节点：接收用户输入，支持自定义变量名
- LLM 节点：调用大语言模型 API，支持模板变量替换
- 代码节点：在沙箱环境中执行 Python 代码，支持函数模式和表达式模式
- RAG 节点：从知识库检索相关内容，支持多种检索模式
- 条件节点：根据条件表达式进行分支跳转
- 输出节点：生成最终输出，支持模板渲染
- 开始/结束节点：标识工作流的起始和结束

**执行流程**：通过 build_langgraph_graph 函数将前端传入的图数据构建为可执行的 StateGraph，然后通过 execute_workflow 函数执行工作流并返回结果。

### 2.3 知识库系统设计

知识库系统实现了完整的文档管理和检索功能：

**文档处理**：支持文档上传、解析和分块，采用可配置的分块大小和重叠长度，保证上下文的连贯性。

**向量化**：使用 BGE 模型生成文本的密集向量和稀疏向量，同时存储到 Milvus 向量数据库中。

**混合检索**：支持三种检索模式：密集检索、稀疏检索和混合检索。混合检索通过分数归一化和加权融合将两种检索结果结合，提高检索准确性。系统还支持可选的重排序（Rerank）功能，进一步优化检索结果。

**检索策略**：实现 candidate_k 初选和 top_k 精选的两阶段检索，通过字符数限制控制检索结果的总长度。

### 2.4 关键技术实现

**沙箱化代码执行**：代码节点在受限环境中执行，通过限制可用内置函数、设置超时机制（25秒）确保安全性。支持两种代码模式：函数模式（包含 def main()）和表达式模式（直接执行代码）。

**动态变量传递**：通过 custom_vars 字典存储各节点的输出，支持变量引用和传递，模板中使用 {{variable}} 语法进行变量替换。

**条件分支**：条件节点根据表达式求值结果选择下一个执行节点，支持复杂的条件逻辑。

**跨域支持**：后端配置 CORS 中间件，支持前端跨域访问。

## 结束语

本文设计并实现了一套基于 React + LangGraph 的可视化 AI 工作流编排系统，通过图形化界面降低了 AI 应用的开发门槛。系统集成了工作流编排、知识库检索、大模型调用、代码执行等多种功能，具有良好的扩展性和灵活性。

实践表明，该系统能够有效支持各类 AI 应用场景的快速构建，提高开发效率。未来工作将进一步优化检索算法、增加更多节点类型、支持分布式执行，并探索多模态数据的处理能力。
