# AI 工作流搭建系统 - 启动说明

## ⚠️ 重要提示

**不要**使用以下命令启动后端：
```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

这会导致 `ModuleNotFoundError: No module named 'app'` 错误！

---

## ✅ 正确启动方式

### 方式1：使用启动脚本（推荐）

**启动后端**：
```bash
cd backend
python run.py
```

**启动前端**（新开终端）：
```bash
cd frontend
npm run dev
```

---

### 方式2：从backend目录启动

```bash
cd backend
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

## 🔧 常见问题排查

### 1. 前端页面一直在加载/转圈

**症状**：工作流列表页面显示"暂无数据"并一直转圈

**原因**：后端服务没有启动或卡死

**解决**：
1. 检查后端是否在运行：`netstat -ano | findstr :8000`
2. 如果没有，重新启动后端：`cd backend && python run.py`
3. 刷新前端页面

---

### 2. API 请求超时

**症状**：控制台显示 `timeout of 10000ms exceeded`

**原因**：后端服务卡死或端口被占用

**解决**：
```bash
# 1. 杀掉所有 Python 进程
taskkill /F /IM python.exe

# 2. 重新启动后端
cd backend
python run.py
```

---

### 3. ModuleNotFoundError: No module named 'app'

**原因**：从错误目录启动 uvicorn

**解决**：
- ✅ 正确：`cd backend && python run.py`
- ❌ 错误：`uvicorn app.main:app` (在项目根目录执行)

---

### 4. 前端端口被占用

**症状**：`Port 3000 is in use, trying another one...`

**解决**：Vite 会自动尝试 3001、3002 等端口，直接访问显示的地址即可

## 🚀 快速重启（遇到问题时的万能方法）

如果前后端都出现问题，按以下步骤重启：

```bash
# 1. 杀掉所有相关进程
taskkill /F /IM python.exe
taskkill /F /IM node.exe

# 2. 启动后端（终端1）
cd c:\Users\admin\Desktop\new_agent\backend
python run.py

# 3. 启动前端（终端2）
cd c:\Users\admin\Desktop\new_agent\frontend
npm run dev
```

## 💡 开发提示

1. **后端修改自动重启**：`run.py` 已配置 `--reload`，修改代码会自动重启
2. **前端热更新**：`npm run dev` 支持热更新，修改代码页面自动刷新
3. **API 调试**：访问 http://localhost:8000/docs 可直接测试 API
4. **数据库**：使用 SQLite，文件位于 `backend/workflow.db`
