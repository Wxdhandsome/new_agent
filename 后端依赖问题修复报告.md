# 后端LangGraph依赖兼容性问题修复报告

## 问题描述

在尝试启动后端FastAPI服务时，出现了LangGraph和pydantic的版本兼容性错误。

## 错误堆栈

```
TypeError: ForwardRef._evaluate() missing 1 required keyword-only argument: 'recursive_guard'
```

## 问题分析

### 根本原因
- **LangGraph 0.0.50** 依赖于 **pydantic v1**
- 项目中安装的是 **pydantic 2.5.3** (v2版本)
- pydantic v1和v2之间存在API不兼容问题，特别是在ForwardRef处理上

### 影响范围
- 后端服务无法启动
- API路由无法加载
- 整个后端系统不可用

## 解决方案

### 方案1: 使用Mock执行（已实施）
1. **修改API文件** (`backend/app/api/workflow.py`)
   - 移除对 `from ..langgraph import execute_workflow` 的直接导入
   - 添加 `mock_execute_workflow` 函数作为临时替代

2. **修改LangGraph模块** (`backend/app/langgraph/__init__.py`)
   - 添加 try-except 包装导入逻辑
   - 如果导入失败，提供mock版本的 `execute_workflow`

### 方案2: 版本降级（备选）
如需真正的LangGraph功能，可以考虑：
- 降级 pydantic 到 v1.x 版本
- 升级 langgraph 到支持 pydantic v2 的版本

## 修复文件清单

1. `backend/app/api/workflow.py` - 使用mock执行函数
2. `backend/app/langgraph/__init__.py` - 添加安全导入包装
